# 🖥️ 나의 서비스 (호스팅)

**나의 서비스**에서 웹·WAS·DB 서비스를 **생성**하고, **배포·관리**할 수 있습니다.  
좌측 메뉴의 **나의 서비스**를 클릭하면 서비스 목록 화면이 열립니다.

---

## 📋 서비스 목록

- **경로**: 나의 서비스 메뉴 → 서비스 목록 화면
- **기능**:
  - **검색·필터**: 서비스 유형(웹/DB/WAS/통합/WAS+DB), 상태 등으로 목록을 필터링할 수 있습니다.
  - **서비스 카드**: 각 서비스의 **도메인**, **서비스 유형**, **상태**, **최근 배포** 정보가 카드에 표시됩니다.
  - **서비스 만들기**: **➕ 서비스 만들기** 버튼을 누르면 서비스 생성 화면으로 이동합니다. 단계별 안내는 [빠르게 시작하기](/getting-started/quick-start)를 참고하세요.
- **카드에서 할 수 있는 것**:
  - 카드를 클릭하면 **서비스 상세** 페이지로 이동합니다.
  - 웹/WAS 서비스 카드에서는 **IDE 열기**, DB 관련 서비스에서는 **SQL 열기** 등 빠른 링크를 사용할 수 있습니다.

---

## ➕ 서비스 생성

1. **나의 서비스** 목록에서 **서비스 만들기** 버튼을 클릭합니다.
2. **플랜 선택**(무료/유료) 후, 유료인 경우 **서비스 유형**을 선택합니다.
3. **서비스 유형에 따른 설정 단계**를 진행합니다.  
   - 웹: WEB 설정(소스 가져오기, 빌드 옵션, 리소스)  
   - WAS: WAS 설정(런타임, 시작 프로그램, 소스)  
   - DB: DB 설정(DB 종류, 리소스)  
   - 통합: WEB 설정 → WAS 설정 → DB 설정  
   - WAS+DB: WAS 설정 → DB 설정  
4. **도메인** 단계에서 서브도메인(및 유료 시 커스텀 도메인)을 설정합니다.
5. **검토 및 생성**에서 내용을 확인한 뒤 **서비스 만들기**(또는 무료/유료 문구 버튼)를 클릭합니다.

단계별 화면 흐름과 옵션 설명은 [빠르게 시작하기](/getting-started/quick-start)를 참고하세요.

---

## 📄 서비스 상세

서비스 **카드**를 클릭하면 해당 서비스의 **상세 페이지**로 이동합니다.  
상세 페이지에는 **탭**으로 구분된 화면이 있으며, **서비스 유형**에 따라 보이는 탭이 달라집니다.

### 공통 탭 (대부분의 서비스 유형)

| 탭 | 설명 |
|----|------|
| **개요** | 서비스 상태, 도메인(서브도메인·커스텀 도메인), 소스 정보(수동 업로드/GitHub/API 등), 리소스 요약. 배포 상태와 최근 배포 정보를 확인할 수 있습니다. |
| **모니터링** | 트래픽, 응답 시간, 방문자 등 모니터링 데이터를 확인할 수 있습니다. |
| **보안** | 방화벽·WAF 설정, 보안 이벤트 등 보안 관련 정보를 확인하고 설정할 수 있습니다. |
| **설정** | 소스 변경, 환경 변수, 리소스·플랜 변경, 확장 설정(로드밸런서, 웹 백업, DB 백업, WAF, DB 분산 등)을 할 수 있습니다. |
| **관리** | 재시작, 소스 업로드(ZIP/TAR), 웹훅 URL 확인, **IDE 열기**, **SQL 열기**(DB/통합 서비스) 등 일상적인 운영 작업을 할 수 있습니다. |

### 서비스 유형별 추가 탭

- **웹 서비스**: **WEB 설정** 탭(웹 소스·빌드·리소스 등)
- **WAS 서비스**: **WAS 설정** 탭(WAS 런타임·소스·환경 변수 등)
- **통합 서비스**: **WEB 설정**, **WAS 설정**, **DB** 탭(DB 연결 정보, DB 전용 설정 등)
- **WAS+DB**: **WAS 설정**, **DB** 탭
- **DB 전용**: **DB** 탭

:::tip
통합 서비스에서는 **DB** 탭에서 DB 연결 정보(호스트, 포트, DB명, 사용자 등)를 확인할 수 있으며, WAS에서 이 정보를 환경 변수로 자동 주입받아 사용할 수 있습니다.
:::

## 📊 서비스 상태

서비스와 배포 진행 상황은 아래와 같은 **상태**로 표시됩니다.

| 상태 | 설명 |
|------|------|
| **준비됨** | 정상 배포가 완료된 상태. 도메인으로 접속해 사용할 수 있습니다. |
| **배포 중** | 배포가 진행 중인 상태. 완료되면 준비됨으로 바뀝니다. |
| **대기 중** | 배포 작업이 대기 중인 상태. |
| **빌드 중** | 소스를 빌드하고 있는 상태(GitHub·공개 저장소 등 빌드 옵션 사용 시). |
| **실패** | 배포 또는 빌드가 실패한 상태. 소스·설정을 확인한 뒤 다시 배포할 수 있습니다. |
| **삭제 실패** | 서비스 삭제 요청이 실패한 상태. |

상태는 목록 카드와 상세 **개요** 탭에서 확인할 수 있습니다.

---

## 🔧 관리 탭에서 할 수 있는 작업

- **재시작**: 서비스를 재시작합니다. 소스나 설정 변경 후 반영이 필요할 때 사용할 수 있습니다.
- **소스 업로드**: ZIP 또는 TAR 파일을 업로드해 배포 소스를 교체할 수 있습니다(수동 업로드 방식인 경우). 업로드 후 재시작이 필요할 수 있습니다.
- **웹훅 URL**: GitHub 등에서 푸시 시 자동 배포를 트리거할 때 사용하는 URL을 확인할 수 있습니다. (민감한 URL이므로 문서에는 실제 값 없이 “웹훅 URL 확인” 수준만 안내)
- **IDE 열기**: 웹 기반 IDE로 해당 서비스(WEB/WAS)의 파일을 편집하고 터미널을 사용할 수 있습니다. 웹·WAS·통합 서비스에서 사용 가능합니다.
- **SQL 열기**: DB 서비스 또는 통합/WAS+DB 서비스의 DB에 접속해 쿼리를 실행하고 관리할 수 있습니다.
