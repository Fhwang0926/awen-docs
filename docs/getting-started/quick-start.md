# 🚀 빠르게 시작하기

첫 서비스를 **나의 서비스** 화면에서 생성하는 방법을 단계별로 안내합니다.  
실제 화면 흐름은 **서비스 생성** 페이지의 퀵 설정(플랜 → 서비스형 → WEB/WAS/DB 설정 → 도메인 → 검토 및 생성)을 따릅니다.

## ✅ 사전 준비

- 회원가입 및 로그인 완료
- **나의 서비스** 메뉴 접근 가능

---

## 1️⃣ 서비스 생성 페이지로 이동

1. 좌측 메뉴에서 **나의 서비스**를 클릭합니다.
2. **추가** 버튼(또는 **서버 생성**)을 클릭해 **서비스 생성** 페이지로 이동합니다.

---

## 2️⃣ 플랜 선택 (유료 사용자만)

유료 플랜 사용자는 **플랜 선택** 단계가 표시됩니다.

| 플랜 | 설명 |
|------|------|
| **🆓 무료 플랜** | 신용카드 없이 바로 시작. 서브도메인 자동 부여. 개인 도메인은 유료에서만 설정 가능. |
| **⭐ 유료 플랜** | 실서비스·풀 기능 권장. 서브도메인 직접 입력, 커스텀 도메인, 로드밸런서·백업·WAF 등 사용 가능. |

:::info
무료 사용자는 이 단계가 생략되고 **서비스형** 선택으로 넘어갑니다.
:::

---

## 3️⃣ 서비스형 선택

어떤 형태로 서비스를 운영할지 **서비스 타입**을 선택합니다.

| 서비스형 | 설명 |
|----------|------|
| **🌐 웹 서버(Frontend)** | 정적·SPA(React, Vue 등) 또는 HTML/CSS/JS. Nginx로 서비스하며 DB는 포함되지 않습니다. |
| **🗄️ DB 서버(전용)** | MySQL, MariaDB, PostgreSQL, MongoDB 등 DB만 독립적으로 운영할 때 선택합니다. |
| **⚙️ WAS 서비스(Backend)** | Python, Node.js, Java 등 백엔드 API만 배포할 때. 프론트·DB 없이 API 서버만 운영합니다. |
| **🔗 통합 서비스(Front + Backend + DB)** | 프론트엔드·WAS·DB를 한 서비스로 구성. 웹 서버·WAS·DB가 자동 구성됩니다. |
| **⚙️🗄️ WAS+DB** | 백엔드 API와 DB만 함께 운영할 때 선택합니다. |

**선택한 서비스형에 따라 다음 단계가 달라집니다.**

- **웹**: WEB 설정 → 도메인 → 검토 및 생성  
- **DB**: DB 설정 → 도메인 → 검토 및 생성  
- **WAS**: WAS 설정 → 도메인 → 검토 및 생성  
- **통합**: WEB 설정 → WAS 설정 → DB 설정 → 도메인 → 검토 및 생성  
- **WAS+DB**: WAS 설정 → DB 설정 → 도메인 → 검토 및 생성  

---

## 4️⃣ WEB 설정 (웹·통합 서비스인 경우)

**배포 소스 & 빌드**와 **서버 구성**을 설정합니다.

### 📂 소스 가져오기

- **GitHub**: GitHub App 연동 후 저장소·브랜치 선택. 연동이 안 되어 있으면 **GitHub 연동** 후 저장소 목록에서 선택합니다.
- **수동 업로드**: 배포할 파일을 zip으로 압축해 업로드합니다. (웹은 빌드 결과물 `dist` 등)
- **공개 GitHub**: 공개 저장소 URL과 브랜치를 입력합니다.
- **배포 후에 설정**: 소스는 나중에 서비스 상세에서 설정합니다.

### 🔧 빌드 옵션 (GitHub/공개 저장소 선택 시)

React, Vue, Angular, Svelte 등 프론트엔드 빌드 방식을 선택할 수 있습니다. 정적만 쓸 경우 **없음**을 선택합니다.

### 💾 리소스 (유료 플랜)

웹 서버 메모리·스토리지 등을 구성 탭에서 설정합니다. 무료 플랜은 고정 스펙입니다.

---

## 5️⃣ WAS 설정 (WAS·통합·WAS+DB인 경우)

백엔드 런타임과 소스를 설정합니다.

### 백엔드 런타임

- **Python**, **Node.js**, **Java** 중 하나를 선택합니다.
- 버전과 **시작 프로그램**(예: `npm start`, `python app.py`, `java -jar app.jar`)을 입력합니다.

### WAS 소스 (통합·WAS+DB인 경우)

- **웹(프론트엔드)과 동일**: 웹과 같은 저장소를 사용합니다.
- **GitHub**: WAS 전용 저장소·브랜치를 선택합니다.
- **공개 GitHub**: WAS용 공개 저장소 URL을 입력합니다.

저장소 내 **백엔드 경로**(예: `backend`, `api`)가 있으면 입력합니다.

---

## 6️⃣ DB 설정 (DB·통합·WAS+DB인 경우)

- **데이터베이스 종류**: MySQL, MariaDB, PostgreSQL, MongoDB 중 선택합니다.
- **리소스**(유료): 메모리·스토리지 할당. 무료는 고정 스펙입니다.

---

## 7️⃣ 도메인

- **무료 플랜**: 서브도메인은 자동 부여됩니다. (개인 도메인은 유료에서만 설정 가능)
- **유료 플랜**:
  - **서브도메인**: 영문 소문자, 숫자, 하이픈만 사용(3–63자). 예: `myapp`. 입력 후 사용 가능 여부가 표시됩니다.
  - **개인 도메인**(선택): 보유한 도메인(예: `example.com`)을 입력하면, 생성 후 DNS에서 CNAME으로 연결하라는 안내가 나옵니다.

---

## 8️⃣ 검토 및 생성

- 상단 **인프라 구성도**와 각 단계에서 입력한 내용을 **최종 확인**합니다.
- **서비스 만들기**(또는 **무료로 지금 만들기** / **프로덕션 서비스 만들기**) 버튼을 클릭합니다.

:::tip 완료
생성이 완료되면 **나의 서비스** 목록에 항목이 추가되고, 상태가 **배포 중** → **준비됨**으로 바뀝니다.
:::

---

## 📋 생성 후 확인

- **나의 서비스**에서 해당 카드를 클릭해 **서비스 상세**로 이동합니다.
- **개요** 탭: 상태, 도메인(서브도메인 또는 커스텀 도메인), 소스, 리소스 요약을 확인합니다.
- **관리** 탭: IDE 열기, SQL 열기(DB/통합 서비스), 로그 등으로 운영합니다.
- **설정** 탭: 소스 변경, 환경 변수, 리소스·플랜 변경을 할 수 있습니다.
