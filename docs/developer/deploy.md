# 배포 가이드

서비스 타입별 소스 연동과 배포 방식을 설명합니다.

## 소스 연동 방식

- **GitHub**: GitHub 저장소를 연동하여 push 시 자동 빌드·배포
- **수동 업로드**: zip 파일을 업로드하여 배포
- **API**: API로 아티팩트를 전달하여 배포

## GitHub 연동 배포

1. 서비스 생성 시 **소스**에서 **GitHub**를 선택합니다.
2. **GitHub 연동** 후 저장소·브랜치를 선택합니다.
3. 저장소에 push하면 자동으로 빌드가 시작되고, 완료 후 배포됩니다.

## 수동 업로드 배포

1. 서비스 생성 시 **소스**에서 **수동 업로드**를 선택합니다.
2. 빌드 결과물(zip) 또는 정적 파일을 압축한 zip을 업로드합니다.
3. 업로드가 완료되면 배포가 진행됩니다.

:::tip
프론트엔드(웹)의 경우 `npm run build` 또는 `yarn build` 결과물을 zip으로 압축해 업로드하면 됩니다.
:::

## 서비스 타입별 참고

- **웹(정적)**: HTML/CSS/JS 또는 SPA 빌드 결과물
- **WAS**: Python, Node.js, Java 등 런타임 선택 후 시작 프로그램·빌드 설정
- **DB**: MySQL, MariaDB, PostgreSQL, MongoDB 등 선택 후 리소스 설정
- **통합**: 웹 + WAS + DB를 한 서비스로 구성

## 환경 변수

서비스 상세 → **설정** 탭에서 환경 변수를 추가할 수 있습니다. 빌드·런타임에서 사용됩니다.
