# 🚀 빠르게 시작하기

**나의 서비스**에서 첫 서비스를 만드는 방법을 화면 흐름에 맞춰 단계별로 안내합니다.  
실제 서비스 생성 화면은 **퀵 설정(Quick Setup)** 단계를 따릅니다.

## ✅ 사전 준비

- 회원가입 및 로그인을 완료합니다.
- 좌측 메뉴에서 **나의 서비스** 메뉴에 접근할 수 있어야 합니다.

---

## 1️⃣ 서비스 생성 화면으로 이동

1. 좌측 메뉴에서 **나의 서비스**를 클릭합니다.
2. **서비스 만들기** 버튼(➕)을 클릭해 **서비스 생성** 페이지로 이동합니다.

---

## 2️⃣ 플랜 선택

먼저 **무료** 또는 **유료** 플랜을 선택합니다.

| 플랜 | 설명 | 생성 단계 |
|------|------|-----------|
| **무료 플랜** | 신용카드 없이 바로 시작. 서브도메인은 자동 부여됩니다. 개인(커스텀) 도메인은 유료 플랜에서만 설정 가능합니다. | **도메인** → **WEB 설정** → **검토 및 생성** 세 단계만 진행됩니다. (웹 서비스만 가능) |
| **유료 플랜** | 실서비스·풀 기능 권장. 서브도메인 직접 입력, 커스텀 도메인, 로드밸런서·백업·WAF·DB 분산 등 사용 가능. | **서비스 유형** 선택 후, 해당 유형에 맞는 설정 단계(WEB/WAS/DB 설정 등) → **도메인** → **검토 및 생성** 순으로 진행됩니다. |

:::info
무료 플랜을 선택하면 **서비스 유형** 단계는 건너뛰고, **도메인** 단계부터 진행됩니다.
:::

---

## 3️⃣ 서비스 유형 선택 (유료 플랜만)

유료 플랜을 선택한 경우, **어떤 형태로 서비스를 운영할지** 서비스 유형을 선택합니다.

| 서비스 유형 | 설명 |
|-------------|------|
| **웹 서버(Frontend)** | 정적·SPA(React, Vue, Angular 등) 또는 HTML/CSS/JS. Nginx로 서비스하며 DB는 포함되지 않습니다. |
| **DB 서버(전용)** | MySQL, MariaDB, PostgreSQL, MongoDB 등 DB만 독립적으로 운영할 때 선택합니다. |
| **WAS 서비스(Backend)** | Python, Node.js, Java 등 백엔드 API만 배포할 때. 프론트·DB 없이 API 서버만 운영합니다. |
| **통합 서비스(Front + Backend + DB)** | 프론트엔드·WAS·DB를 한 서비스로 구성. 웹 서버·WAS·DB가 자동 구성됩니다. 통합 서비스에서는 같은 도메인에서 `/api` 요청이 WAS로 자동 전달됩니다. |
| **WAS+DB** | 백엔드 API와 DB만 함께 운영할 때 선택합니다. 프론트엔드 없이 API 서버와 데이터베이스만 운영할 때 적합합니다. |

선택한 서비스 유형에 따라 다음 단계가 달라집니다.

- **웹**: WEB 설정 → 도메인 → 검토 및 생성  
- **DB**: DB 설정 → 도메인 → 검토 및 생성  
- **WAS**: WAS 설정 → 도메인 → 검토 및 생성  
- **통합**: WEB 설정 → WAS 설정 → DB 설정 → 도메인 → 검토 및 생성  
- **WAS+DB**: WAS 설정 → DB 설정 → 도메인 → 검토 및 생성  

---

## 4️⃣ WEB 설정 (웹·통합 서비스인 경우)

**배포 소스**와 **빌드 방식**, 필요 시 **리소스**를 설정합니다.

### 소스 가져오기

코드를 어디서 가져올지 선택합니다.

| 옵션 | 설명 |
|------|------|
| **GitHub 저장소** | GitHub App 연동 후 저장소·브랜치를 선택합니다. 연동이 되어 있지 않으면 **GitHub 연동** 안내가 나옵니다. |
| **수동 업로드** | 배포할 파일을 ZIP(또는 TAR, TAR.GZ)으로 압축해 업로드합니다. 웹은 빌드 결과물(예: `dist`)을 압축해 올리면 됩니다. (최대 200MB) |
| **공개 저장소 URL** | GitHub 공개 저장소 URL과 브랜치를 입력합니다. 로그인 없이 배포할 수 있습니다. |
| **배포 후에 설정** | 서버만 먼저 만들고, 소스는 나중에 서비스 상세의 **관리** 탭에서 업로드하거나 설정합니다. |

### 빌드 옵션 (GitHub·공개 저장소 선택 시)

React, Vue, Angular, Svelte 등 프레임워크를 선택하면 저장소 소스를 자동으로 빌드해 배포합니다. 정적 파일만 쓸 경우 **빌드 없음(HTML, CSS, JS 그대로)** 을 선택합니다.

### 리소스 (유료 플랜)

웹 서버 메모리·스토리지는 **구성 설정** 탭에서 설정할 수 있습니다. 무료 플랜은 고정 스펙(예: 메모리 2GB, 스토리지 5GB)이 적용됩니다.

---

## 5️⃣ WAS 설정 (WAS·통합·WAS+DB인 경우)

백엔드 **런타임**과 **소스**를 설정합니다.

### 백엔드 런타임

- **Python**, **Node.js**, **Java** 중 하나를 선택하고, 사용할 **버전**을 선택합니다.
- **시작 프로그램**에 구동 시 실행할 명령을 입력합니다.  
  예: `npm start`, `python app.py`, `java -jar app.jar`  
  비워두면 런타임 기본값이 사용됩니다.

### WAS 소스 (통합·WAS+DB인 경우)

- **웹(프론트엔드)과 동일**: 웹과 같은 저장소를 사용합니다.
- **GitHub**: WAS 전용 저장소·브랜치를 선택합니다.
- **공개 GitHub**: WAS용 공개 저장소 URL을 입력합니다.

저장소 안에 **백엔드 경로**(예: `backend`, `api`)가 있으면 해당 경로를 입력합니다.

---

## 6️⃣ DB 설정 (DB·통합·WAS+DB인 경우)

- **데이터베이스 종류**: MySQL, MariaDB, PostgreSQL, MongoDB 중에서 선택합니다.
- **리소스**(유료): DB 메모리·스토리지를 설정합니다. 무료 플랜은 고정 스펙이 적용됩니다.

---

## 7️⃣ 도메인

- **무료 플랜**: 서브도메인은 **자동 생성**됩니다. 개인 도메인은 유료 플랜에서만 설정할 수 있습니다.
- **유료 플랜**:
  - **서브도메인**: 영문 소문자, 숫자, 하이픈만 사용 가능(3~63자). 예: `myapp`. 입력 후 **사용 가능 여부**를 확인할 수 있습니다. **자동 생성** 버튼으로 자동 입력할 수도 있습니다.
  - **개인(커스텀) 도메인**(선택): 보유한 도메인(예: `example.com`)을 입력하면, 생성 후 DNS에서 CNAME 등으로 연결하라는 안내가 표시됩니다.

:::tip
커스텀 도메인 연결 시 DNS 제공업체(가비아, 후이즈, Cloudflare 등)에서 CNAME 레코드를 추가하고, 안내된 값으로 설정하면 됩니다. SSL 인증서는 서비스 생성 시 자동으로 발급됩니다.
:::

---

## 8️⃣ 검토 및 생성

- 상단 **인프라 구성도**와 각 단계에서 입력한 내용을 **최종 확인**합니다.
- **서비스 만들기**(무료: **무료로 지금 만들기**, 유료: **프로덕션 서비스 만들기**) 버튼을 클릭합니다.

:::tip 완료
생성이 완료되면 **나의 서비스** 목록에 항목이 추가되고, 상태가 **배포 중** → **준비됨**으로 바뀝니다. 카드를 클릭하면 서비스 상세 페이지로 이동합니다.
:::

---

## 📋 생성 후 확인

- **나의 서비스**에서 해당 **카드**를 클릭해 **서비스 상세**로 이동합니다.
- **개요** 탭: 상태, 도메인(서브도메인 또는 커스텀 도메인), 소스, 리소스 요약을 확인할 수 있습니다.
- **관리** 탭: **IDE 열기**, **SQL 열기**(DB/통합 서비스), **재시작**, 소스 업로드, 웹훅 URL 확인 등으로 운영할 수 있습니다.
- **설정** 탭: 소스 변경, 환경 변수, 리소스·플랜 변경을 할 수 있습니다.
